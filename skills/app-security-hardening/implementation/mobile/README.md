# Mobile Implementation

Mobile implementation docs explain how app teams build the controls described by `controls/`, `platforms/`, and `languages/`.

| File | Scope | Linked platform/language docs |
| --- | --- | --- |
| `android.md` | Android manifest, storage, network config, release behavior. | `../../platforms/mobile/android.md`, `../../languages/kotlin-java.md` |
| `ios.md` | iOS Keychain, ATS, URL handling, release behavior. | `../../platforms/mobile/ios.md`, `../../languages/swift.md` |
| `flutter.md` | Flutter platform channels, packages, release checks, storage. | `../../platforms/mobile/flutter.md`, `../../languages/dart.md` |
| `react-native.md` | Native modules, JS bundle, OTA, secure storage bridge. | `../../platforms/mobile/react-native.md`, `../../languages/typescript.md` |

Required linked updates: changes here must update or verify the relevant `../../controls/` docs and review checklists.
