# Engineering Philosophy

`intelligence/engineering/philosophy/` 保存可跨工程領域重用的原始思想來源與認知框架。這一層回答「這個治理或 workflow 背後的工程哲學是什麼」，不直接定義 AI runtime 的可執行規則。

## 目前條目

| 文件 | 核心思想 | Governance translation |
| --- | --- | --- |
| [`musk-five-step-algorithm.md`](musk-five-step-algorithm.md) | Question requirements, delete, simplify, accelerate, automate last | [`governance/ai-runtime-governance/five-step-ai-governance.md`](../../../governance/ai-runtime-governance/five-step-ai-governance.md) |

## 放什麼

- 原始 engineering philosophy。
- First-principles reasoning framework。
- 可跨 domain 套用的 complexity reduction、automation timing、system design 思想。
- 後續治理文件可以引用的 source philosophy。

## 不放什麼

- 可執行 policy；放到 `enforcement/` 或 governance translation。
- AI runtime gate；放到 `governance/ai-runtime-governance/`。
- 單一 workflow 的 checklist；放到 `workflow/`。
- 百科背景或人物傳記；只保留與工程判斷相關的思想。
