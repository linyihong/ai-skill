# narrow_mobile

Use when the UI must survive the smallest supported phone-like constraints.

## Governs

- Single-column content priority.
- Minimum readable density.
- Primary action reachability.
- Horizontal overflow risk.
- Full-height scroll-root behavior on short viewports.

## Fixture Guidance

Choose any representative viewport that stresses both width and height. The fixture can be a device preset, but the contract is the narrow constraint, not the device name.
