# Cursor 使用說明

本檔只記錄 Cursor 與其他 agent 工具不同的地方。通用 bootstrap 與 obligation 來源一律是 [`CORE_BOOTSTRAP.md`](../../CORE_BOOTSTRAP.md) 與 [`runtime/core-bootstrap.yaml`](../../runtime/core-bootstrap.yaml)。

## Thin Entry Point

Cursor 的 repo 入口是 `.cursor/rules/ai-skill-bootstrap.mdc`（`alwaysApply: true`）。它必須保持 thin pointer，只指向：

1. [`CORE_BOOTSTRAP.md`](../../CORE_BOOTSTRAP.md)
2. [`runtime/core-bootstrap.yaml`](../../runtime/core-bootstrap.yaml)

不要在 Cursor rule 或本 adapter 複製 Bootstrap Receipt、Cognitive Mode、close-loop、goal ledger 或 runtime phase 的細節；需要時從 canonical source 讀取。

## Cursor 差異

- `.cursor/rules/ai-skill-bootstrap.mdc` is the project Cursor entry. Keep it as the only project-level Ai-skill `.mdc` generated by `init-project`.
- Cursor global rules and project rules merge, so extra project-rule `.mdc` files can make validation depend on Cursor-only discovery.
- Project-specific rule bodies should live in `<PROJECT_ROOT>/.ai-skill/project/rules/` and be discovered through `<PROJECT_ROOT>/.ai-skill/project/README.md`, not through separate Cursor project-rule `.mdc` files.
- `.cursor/hooks.json` 不應讓 hook 靜默修改檔案、關閉 goal 或自動 commit。但自 2026-06-05 起，Cursor 端**有** pre-tool 機械 gate（不再只是提醒）：見下方 `preToolUse`。
- `ai-skill init-project --tools cursor` 產生的 project hook 包含三個：`sessionStart`（close-out reminder）、`preToolUse`（機械 gate）、`stop`（close-out check），皆呼叫 repo-local `ai-skill hooks run <event>`。
- **`preToolUse` 機械 gate（mechanical L3，與 Claude PreToolUse 對等）**：偷跑非-Read 工具且 bootstrap receipt 未滿足、或 workflow `primary_source` 未讀時攔截。block 用 Cursor native **`exit 0` + stdout `{"permission":"deny",user_message,agent_message}`**（Cursor 也接受 exit 2，但採 JSON 以帶理由）；allow = 空 stdout。`detectPreToolUseHost` 以 `cursor_version` 分流，gate 邏輯與 Claude 共用（`renderPreToolUseDecision` per-host）。read-type 工具（read_file/list_dir/grep/glob_file_search/codebase_search）一律放行，故 agent 仍可讀要遵從的檔；`beforeReadFile` 不接（避免雙重攔 read）。`AI_SKILL_REPO` 無法解析時此 gate **fail-open（exit 0）**，不會擋掉所有工具。
- **`stop` close-out check**：Cursor stop 不能靠 exit 2 loop back；缺少 Bootstrap Receipt、compact `Cognitive:` / full `### Cognitive Mode 報告`，或必要的 `### Project Git Report` 時，hook runner 一次彙整缺項，輸出 `followup_message` 並 exit 0（`failClosed: true`：`AI_SKILL_REPO` 缺失時 fail-closed，與 preToolUse 相反）。格式與枚舉仍只在 canonical bootstrap sources。
- 多資料夾工作區可同時打開業務專案與本 repository，讓 agent 直接讀 canonical source。

## 配置邊界

Cursor-specific 路徑、hooks、UI 行為與 workspace 操作留在本檔或 `.cursor/` 設定。跨工具規則放回 `enforcement/`，runtime contract 放回 `runtime/core-bootstrap.yaml`。
