# Small / Constrained Model Workflows

Small 或 constrained profile 適合低風險、低 ambiguity、可用 checklist 驗證的工作。它不代表能力低，只表示預設 context budget 與 autonomy 要更保守。

## Workflow Shape

1. 先讀 index、summary、registry 或 checklist。
2. 明確列出 deferred source。
3. 只做 bounded edit、format、lookup 或 checklist execution。
4. 任何 canonical source 修改前升級到 source-backed。
5. 用 validation signal 結束，不做 global claim。

## 適合任務

- 快速定位 source。
- 套用已驗證格式。
- 小型 README link / typo 修正。
- 已有 checklist 的 low-risk closeout。

## Escalation

出現 contradiction、dirty generated surface、source-of-truth ambiguity、commit/push、runtime DB、rules 或 user correction 時，立即升級到 source-backed / validation-heavy。
